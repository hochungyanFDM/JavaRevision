1. What is Java?

Java is a high-level, object-oriented programming language. It runs on the JVM, so the same Java code can run on different operating systems.

Example: write once, run anywhere.

2. What is OOP?

OOP means Object-Oriented Programming. It structures code around objects, which contain data and methods.

Example: Car object has data like color, and methods like drive().

3. What are the 4 pillars of OOP?

The 4 pillars are:

Encapsulation, abstraction, inheritance, and polymorphism.

4. What is encapsulation?

Encapsulation means hiding data inside a class and controlling access using methods.

class Person {
    private String name;

    public String getName() {
        return name;
    }
}

name is private, so other classes cannot directly change it.

5. What is abstraction?

Abstraction means showing only important details and hiding unnecessary implementation.

Example: when you use car.start(), you do not need to know how the engine works internally.

6. What is inheritance?

Inheritance means one class can reuse fields and methods from another class.

class Animal {
    void eat() {
        System.out.println("Eating");
    }
}

class Dog extends Animal {
}

Dog can use eat() from Animal.

7. What is polymorphism?

Polymorphism means the same method name can behave differently depending on the object.

class Animal {
    void sound() {
        System.out.println("Animal sound");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("Bark");
    }
}

A Dog has its own version of sound().

## 8. Interface vs Abstract Class

An **interface** is a **contract or capability**. It defines what a class must be able to do, but it does not usually provide the full implementation.

Example:

```java id="lrtynk"
interface Flyable {
    void fly();
}
```

Any class that implements `Flyable` must provide the `fly()` method.

```java id="r2tesm"
class Bird implements Flyable {
    public void fly() {
        System.out.println("Bird flies with wings");
    }
}

class Plane implements Flyable {
    public void fly() {
        System.out.println("Plane flies with engines");
    }
}
```

A `Bird` and a `Plane` are not the same type of object, but both have the same ability: they **can fly**.

So the easy memory is:

```text id="re4rc6"
Interface = can do
```

---

An **abstract class** is a **partially built parent class**. It is used when different child classes are related and share some common behaviour, but still need to define some details themselves.

Example:

```java id="keyvju"
abstract class Animal {
    abstract void sound();

    void sleep() {
        System.out.println("Sleeping");
    }
}
```

Here, `Animal` gives every animal a shared `sleep()` method, but it does not define `sound()` because each animal makes a different sound.

```java id="kp8wgp"
class Dog extends Animal {
    void sound() {
        System.out.println("Bark");
    }
}

class Cat extends Animal {
    void sound() {
        System.out.println("Meow");
    }
}
```

A `Dog` and a `Cat` are both animals, so they can share the `Animal` parent class.

So the easy memory is:

```text id="fpcyqa"
Abstract class = is a type of
```

---

### Main Difference

A class can only **extend one class**, but it can **implement many interfaces**.

Example:

```java id="jtfefs"
class Bird extends Animal implements Flyable {
    void sound() {
        System.out.println("Tweet");
    }

    public void fly() {
        System.out.println("Bird is flying");
    }
}
```

This means:

```text id="89c889"
Bird is an Animal.
Bird can Fly.
```

`Animal` describes what the object **is**.
`Flyable` describes what the object **can do**.

---

### When to Use Each

Use an **interface** when you want to define an ability that different classes can share.

Example:

```text id="kpdr5i"
Bird can fly.
Plane can fly.
Drone can fly.
```

They are different types of objects, but they all share the same capability.

Use an **abstract class** when classes are from the same family and share common code.

Example:

```text id="k0d105"
Dog is an Animal.
Cat is an Animal.
Bird is an Animal.
```

They are all related, so an abstract parent class makes sense.

---

### Interview Answer

An interface defines a contract or capability. It tells a class what methods it must implement. It is useful when different classes share the same ability, even if they are not closely related.

An abstract class is a partially implemented parent class. It can contain both abstract methods and normal methods. It is useful when related classes share common behaviour, but still need to implement some methods differently.

The key difference is that a class can implement multiple interfaces, but it can only extend one class. In simple terms, an interface means **“can do”**, while an abstract class means **“is a type of”**.


9. Overloading vs overriding?

Overloading means same method name, different parameters, same class.

void add(int a, int b) {}
void add(int a, int b, int c) {}

Overriding means child class replaces parent method.

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Bark");
    }
}
10. == vs .equals()?

== compares memory reference for objects.

.equals() compares actual content, if implemented properly.

String a = new String("hi");
String b = new String("hi");

System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true
11. String vs StringBuilder vs StringBuffer?

String is immutable. Once created, it cannot be changed.

StringBuilder is mutable and faster, but not thread-safe.

StringBuffer is mutable and thread-safe, but slower.

Use StringBuilder for most normal string modifications.

12. Array vs ArrayList?

Array has fixed size.

int[] nums = new int[3];

ArrayList can grow and shrink.

ArrayList<Integer> nums = new ArrayList<>();
nums.add(10);

Use ArrayList when the size can change.

13. HashMap vs HashSet?

HashMap stores key-value pairs.

Map<String, Integer> ages = new HashMap<>();
ages.put("Justin", 33);

HashSet stores unique values only.

Set<String> names = new HashSet<>();
names.add("Justin");

Easy memory:

HashMap = key to value
HashSet = no duplicates

14. What are hashCode() and equals() used for?

They are used to compare objects, especially in HashMap and HashSet.

equals() checks if two objects are logically the same.

hashCode() gives a number used to place the object into a hash-based collection.

Rule: if two objects are equal, they must have the same hashCode.

15. What is a constructor?

A constructor is a special method used to create and initialise an object.

class Person {
    String name;

    Person(String name) {
        this.name = name;
    }
}

When you do:

Person p = new Person("Justin");

the constructor runs.

16. What is static?

static means the field or method belongs to the class, not a specific object.

class Calculator {
    static int add(int a, int b) {
        return a + b;
    }
}

Call it like this:

Calculator.add(2, 3);

No object needed.

17. What is final?

final means cannot be changed.

For variable:

final int age = 33;

You cannot assign a new value.

For method: cannot be overridden.

For class: cannot be inherited.

18. What is exception handling?

Exception handling means dealing with errors without crashing the program.

try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("Cannot divide by zero");
}

It lets the program handle problems safely.

19. Checked vs unchecked exceptions?

Checked exceptions must be handled at compile time.

Example:

IOException
SQLException

Unchecked exceptions happen at runtime.

Example:

NullPointerException
ArithmeticException
IndexOutOfBoundsException

Easy memory:

Checked = compiler forces you to handle it
Unchecked = runtime problem

20. What is garbage collection?

Garbage collection is the JVM process that automatically removes unused objects from memory.

Person p = new Person();
p = null;

Now the Person object has no reference, so it is eligible for garbage collection.
